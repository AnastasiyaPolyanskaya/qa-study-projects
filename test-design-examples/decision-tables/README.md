# Таблица принятия решений — примеры

Раздел содержит примеры применения техники *Decision Tables* (таблиц принятия решений) для выявления логики системы, комбинаций условий и ожидаемых действий.

## Что такое решающие таблицы

Решающая таблица — это структурированный способ представить:
- условия,
- их возможные значения,
- комбинации условий,
- и соответствующие действия (результаты).

Техника помогает:
- находить пропущенные комбинации,
- выявлять ошибочную бизнес-логику,
- сокращать количество тестов без потери покрытия.

---

## Пример: Авторизация по логину и паролю

**Условия:**
1. Логин корректный?
2. Пароль корректный?

**Действие:**
- Успешный вход / Ошибка авторизации

### Таблица решений

| №  | Корректный логин | Корректный пароль | Ожидаемый результат           |
|----|-------------------|--------------------|--------------------------------|
| 1  | Да                | Да                 | Успешная авторизация           |
| 2  | Да                | Нет                | Ошибка: неверный пароль        |
| 3  | Нет               | Да                 | Ошибка: логин не найден        |
| 4  | Нет               | Нет                | Ошибка: логин не найден        |

## Пример: Проверка скидки в корзине

**Условия:**
1. Промокод введён?
2. Промокод валиден?
3. Сумма корзины больше 1000?

**Действие:**
- Применить скидку / Не применять

### Таблица решений

| № | Промокод введён | Валиден | >1000 | Результат                    |
|---|------------------|---------|-------|-------------------------------|
| 1 | Нет              | —       | —     | Скидка не применяется        |
| 2 | Да               | Нет     | —     | Ошибка: неверный промокод    |
| 3 | Да               | Да      | Нет   | Скидка не применяется        |
| 4 | Да               | Да      | Да    | Скидка применяется           |

## Назначение раздела
