# Граничные значения — примеры

Раздел содержит примеры применения техники анализа граничных значений при тестировании форм, числовых полей и пользовательских сценариев.

## Что такое граничные значения

Анализ граничных значений — это техника, основанная на тестировании крайних точек диапазонов.  
Большинство дефектов возникает именно на границах, поэтому важно проверять значения около пределов.

Границы включают:
- минимальное значение
- максимальное значение
- значение сразу ниже минимума
- значение сразу выше минимума
- значение сразу ниже максимума
- значение сразу выше максимума

## Пример: Поле «Возраст» (диапазон 1–120)

### Проверяемые значения
- 0 — ниже нижней границы (некорректно)
- 1 — нижняя граница (корректно)
- 2 — соседнее после нижней границы (корректно)
- 119 — соседнее перед верхней границей (корректно)
- 120 — верхняя граница (корректно)
- 121 — выше верхней границы (некорректно)

### Ожидаемые результаты
- 0 → ошибка валидации
- 1 → успешный ввод
- 2 → успешный ввод
- 119 → успешный ввод
- 120 → успешный ввод
- 121 → ошибка валидации

## Пример: Поле «Цена» (допустимые значения 1–9999)

Проверяем:
- 0
- 1
- 2
- 9998
- 9999
- 10000
