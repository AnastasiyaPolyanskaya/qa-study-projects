# Классы эквивалентности — примеры

Раздел содержит примеры применения техники классов эквивалентности при тестировании форм, полей ввода и пользовательских сценариев.

## Что такое классы эквивалентности

Классы эквивалентности — это разбиение входных данных на группы, внутри которых система обрабатывает значения одинаково.  
Из каждой группы берётся один представитель — это уменьшает количество тестов, сохраняя максимальное покрытие.

## Примеры в данном разделе

- Поле ввода возраста  
- Поле Email  
- Поле пароля  
- Поле номера телефона  
- Числовые поля (цена, количество, рейтинг и др.)

## Пример: Поле «Возраст» (1–120)

### Валидные классы
- 1–120

### Невалидные классы
- Меньше 1  
- Больше 120  
- Символы или текст  
- Пустое значение  

### Тесты
- 25 (валидный представитель)  
- 0 (невалидный)  
- 150 (невалидный)  
- "abc" (невалидный)  
- "" (пустая строка)
